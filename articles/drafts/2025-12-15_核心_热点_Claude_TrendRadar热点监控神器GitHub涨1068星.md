## MCP集成：让Claude帮你分析热点

TrendRadar V3.0加了一个MCP功能。
MCP是Model Context Protocol的缩写，简单说就是让AI能读懂你的数据。

传统用法是这样的流程：
TrendRadar抓热点，推送到你手机，你自己一条条看，判断哪个值得写。

MCP加持后，流程变了：
TrendRadar抓热点，Claude直接读取数据，你自然语言提问，Claude分析给你答案。

我给Claude Code配置好TrendRadar的MCP服务器，然后问它：
```markdown
"昨天知乎有哪些AI相关的热点？帮我找出最适合写公众号的那个"
```

Claude扫了一遍TrendRadar抓的数据，给我推荐了3个热点，还告诉我为什么适合写。

话题热度高，知乎热榜前10。

## 手把手教你装

### Docker部署（最简单的方式）

如果你装了Docker，一条命令搞定：
```markdown
docker run -d --name trend-radar \
  -v ./config:/app/config:ro \
  -v ./output:/app/output \
  -e FEISHU_WEBHOOK_URL="你的webhook" \
  wantcat/trendradar:latest
```

config目录放配置文件，output目录存热点数据。
FEISHU_WEBHOOK_URL是飞书机器人的webhook地址。
你可以换成企业微信、钉钉、Telegram，都支持。

### 配置关键词

config/frequency_words.txt里配置你关心的关键词：
```markdown
# 科技热点
AI
Claude
Cursor
+发布
!广告

# 财经热点
A股
比特币
+暴涨
!预测
```

"+"开头的是必须包含的词，"!"开头的是过滤词。
空行用来分组，每组独立筛选。

### 配置MCP（可选，但老金建议你装）

如果你用Claude Code或Cursor，这个MCP功能真的值得配。

首先启动TrendRadar的HTTP服务（默认端口3333）。
然后在Claude Code里添加MCP服务器：
```markdown
claude mcp add --transport http trendradar http://localhost:3333/mcp
```

验证连接：
```markdown
claude mcp list
```

看到trendradar就说明配置成功了。

现在你可以在Claude Code里直接问TrendRadar的数据。

比如问它：
```markdown
"查询昨天的知乎AI热点"
"分析一下比特币最近的讨论热度趋势"
"帮我找出最适合写公众号的3个热点"
```

Claude会自动调用TrendRadar的13种分析工具。
包括趋势追踪、情感分析、相似检索等。

## 真实的坑（必须说）

TrendRadar虽然好用，但也有坑：

### 关键词配置有门槛

你得知道怎么配"必须词"和"过滤词"。
比如搜"AI"会刷出一堆广告，你得加"!广告"、"!推广"这种过滤词。

刚开始我配得不对，每天推送200条热点，其中180条是垃圾信息。
第一天收到推送，我翻了20分钟才找到3条有用的。
当时想着这工具不行啊。

后来仔细研究了配置文档，发现是我关键词配置太宽泛了。
调整了两天，现在推送的热点基本都有价值。
准确率能到90%。

**避坑方法**：先从少量关键词开始，慢慢调优。

### MCP配置不够傻瓜

TrendRadar的MCP文档写得有点简略，我配了1小时才搞定。
主要是HTTP端口和传输模式要对应上，否则Claude连不上。

**避坑方法**：严格按文档来，端口用默认的3333，传输模式选HTTP。

### 部分平台抓取不稳定

有些平台会反爬虫，TrendRadar偶尔会抓不到数据。
比如微博热搜，有时候会返回空结果。

**影响**：问题不大，35个平台总有其他的能抓到数据。
