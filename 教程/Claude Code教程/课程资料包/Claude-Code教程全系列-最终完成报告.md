## 五、核心改进方法论

### 5.1 批判性思维（Linus式）

**应用的Linus三问**：
1. "这是真实问题还是想象的？" → 只解决学员真实困惑
2. "有没有更简单的方法？" → 先5分钟快速开始，再深入
3. "这会破坏什么？" → 新建文档，不破坏原有内容

**Good Taste原则**：
- ❌ 坏教程：先讲理论，学员困惑
- ✅ 好教程：先实战见效，再讲原理

**应用到每个教程**：
- 安装指南：先安装验证，再IDE集成
- MCP：先配Filesystem看效果，再讲JSON-RPC
- Hooks：先配最简单Hook，再讲6种类型
- Skills：先创建简单Skill，再讲复杂架构
