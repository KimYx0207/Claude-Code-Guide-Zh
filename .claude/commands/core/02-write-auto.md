## ⚡ 执行流程（完全自动，无需人工干预）

### 步骤0：选题过滤预检（V7.2新增，自动执行）

**在调研前先判断选题分类和时效性**：

1. **检查是否为核心工具类**（品牌词匹配）
2. **检查时效性**（热点期 vs 常青期）
3. **如涉及具体新产品**：用WebSearch确认发布时间
   - 一个月内发布 → 热点期，快写优先
   - 一个月以上 → 常青期，打磨质量

**选题分类影响后续策略**：
| 分类 | 时效性 | 写作策略 |
|------|--------|----------|
| 核心工具类 | 热点期 | 🔥 追热点，快速产出，24小时内发布 |
| 核心工具类 | 常青期 | 📚 做教程，深度打磨，注重质量 |
| 泛AI话题类 | 热点期 | 🎯 博爆款！配合热点传播 |
| 泛AI话题类 | 常青期 | 🤔 谨慎写，需要独特角度 |

根据热点内容自动判断文章类型：

| 热点特征 | 判断类型 | 使用模板 |
|---------|---------|---------|
| 新产品/新版本/新功能 | 测评类 | `templates/测评类文章模板.md` |

**判断规则**：
```
IF 热点包含 "发布/更新/上线/新版" → 测评类
ELSE IF 热点包含 "降价/优惠/折扣/限时/促销" → 降价类
ELSE → 经验故事类
```

### 步骤4：自动写作（自动执行）

**🔴 写作前必须读取格式规范**：
```
Read(".claude/skills/gongzhonghao-writer/prompts/format-rules.md")
```

**⚠️ 格式硬性要求（必须遵守）**：
1. **不要用`---`分隔线** → 直接用`##`标题分隔章节
2. **标题后不空行** → `## 标题\n正文`（紧贴）
3. **代码块前不空行** → `正文：\n```markdown`（紧贴）
4. **序号用顿号** → `1、2、3、`（不用`1.`或`-`）
5. **禁用Markdown表格** → 用列表替代

**写作规范**：

1、**字数要求**：1500-4000字

2、**语言风格**：
   - 老金风格：产品经理视角，不会代码的人设
   - 大白话讲技术，像和朋友聊天
   - 禁止AI腔：不用"首先/其次/总之"，不用"非常/十分"

3、**时间线具体化**：
   - ✅ "周五凌晨"、"昨晚11点"、"今早起来"
   - ❌ "最近"、"前段时间"、"有一次"

4、**案例具体化**：
   - ✅ 有具体产品名、具体数据、具体截图
   - ❌ "某个工具"、"效果很好"、"速度很快"

5、**必须有踩坑**：
   - 至少3个真实问题/限制
   - 每个坑有具体表现
   - 有解决方法或影响评估

6、**口语化表达**：
   - ✅ "玩/扔/牛逼/真香/绝了"
   - ❌ "测试/发现/非常好/值得推荐"

### 步骤6：真实性验证（自动执行）

**V7.1 5维度检查**：

```
□ 时间线具体化：有"周五/昨晚"，没有"最近"
□ 案例具体化：有具体名称和数据
□ 踩坑经历：至少3个真实问题
□ 俚语口语化：有"玩/试"，没有"测试/非常好"
□ 逗号断句：句子不超过20字
```

**不通过自动修正**：
- 自动将"最近"替换为具体时间
- 自动将"测试"替换为"玩/试"
- 自动将"非常好"替换为"好用/香"
- ⚠️ V7.1注意：避免"惊了/麻了"等情绪词（数据显示负相关）
- 自动补充踩坑内容（如果不足3个）

### 步骤8：自动配图（自动执行）✨

**配图策略**：
- 文章长度 ≤ 1500字：1张封面图
- 文章长度 1500-3000字：3张配图（开头+中间+结尾）
- 文章长度 > 3000字：5张配图（均匀分布）

**配图流程**：
```
1、分析文章关键词（标题+二级标题+加粗词）
2、调用Unsplash搜索高质量配图
3、在合适位置插入Markdown图片语法
4、添加图片来源标注
```

**调用工具**：
```
mcp__mcp-router__search_images(
    query="[关键词英文化]",
    per_page=5,
    orientation="landscape"
)
```

**插入格式**：
```markdown
![](图片URL)
```
**格式要求**：
- alt文字留空（方括号内不写字）
- 不加图片说明/署名
- 图片上下各空一行
- URL处理成干净格式：`https://images.unsplash.com/photo-{ID}?w=800`
  （删除原始URL的`?`后所有参数，只保留`?w=800`）

**配图位置**：
- 第1张：开头钩子之后（第1个---分隔线之后）
- 第2张：核心章节中间（数据对比/实战案例之后）
- 第3张：总结升华之前（参考来源之前）

## ⚠️ 异常处理

```

1、提示用户检查网络连接
2、提供手动输入信息的选项
3、使用WebSearch/WebFetch作为最后备选
```

```
如果连续3次修正后仍不通过：
1、输出当前版本和问题维度
2、提示用户手动调整
3、保存为草稿（带"_draft"后缀）
```

### 模板加载失败
```
如果模板文件不存在：
1、使用内置默认结构
2、提示用户检查模板文件
```
