# 直播教程写作规范 V3.0

当用户提到"直播教程"时，自动应用以下规范。

**参考标准**：`教程/Claude Code教程/课程资料包/01-环境与安装/Claude-Code完整安装指南.md`

---

## 一、核心原则

### 1.1 小白友好原则（最重要！）

**每个操作必须回答三个问题**：
1. **这是什么？** - 用生活类比解释概念
2. **为什么要做？** - 说明目的和价值
3. **怎么操作？** - 具体步骤+预期结果

**示例**：
```markdown
### 安装Node.js

**这是什么？**
Node.js就像一个"翻译官"，让你的电脑能够运行JavaScript代码。就像你需要安装微信才能聊天一样，需要安装Node.js才能运行Claude Code。

**为什么要安装？**
Claude Code是用JavaScript写的，没有Node.js就无法运行。这是必装的基础环境。

**操作步骤：**
1. 打开Node.js官网：https://nodejs.org/
2. 下载LTS版本（长期支持版，更稳定）
3. 双击安装包，一路点"Next"

**验证安装：**
```bash
node --version
# 预期输出：v22.x.x（数字可能不同，有输出就是成功）
```
```

### 1.2 资料搜索验证原则（必须执行！）

**写教程前必须WebSearch验证以下信息**：
1. ✅ 软件最新稳定版本号
2. ✅ 官方推荐的安装方式
3. ✅ 已知的兼容性问题
4. ✅ 常见错误和解决方案

**验证流程**：
```
步骤1：WebSearch "[软件名] latest stable version 2025"
步骤2：WebSearch "[软件名] official installation guide"
步骤3：WebSearch "[软件名] common errors solutions"
步骤4：记录信息来源URL
```

**信息标注格式**：
```markdown
> 📌 **信息来源**：[官方文档](URL) | 验证日期：2025-12-XX
```

### 1.3 正确性验证原则（每步必须）

**每个操作步骤后必须提供验证方法**：
```markdown
**验证方法：**
```bash
# 验证命令
命令
# 预期输出：具体预期结果
```

**如果失败：**
可能原因1：xxx
解决方案：xxx
```

## 二、文档结构规范

### 2.1 文档头部（必须）

```markdown
# [教程标题]：[副标题描述核心价值]

> **课程信息**
> - **预计学时**：X-X小时
> - **难度等级**：⭐ 零基础入门 / ⭐⭐ 入门级 / ⭐⭐⭐ 进阶
> - **更新日期**：YYYY年MM月
> - **适用版本**：[软件/工具版本号]

---
```

### 2.2 学习目标（推荐）

```markdown
## 📚 本课学习目标

完成本课学习后，你将能够：

1. **理解XX的核心价值**：掌握XX与传统工具的本质区别
2. **搭建完整开发环境**：在任意操作系统上成功安装XX
3. **完成XX配置**：获取并正确配置XX
4. **验证环境可用性**：通过Hello World测试确认正常工作
5. **掌握故障排查技能**：独立解决90%的常见问题
```

### 2.3 术语表（零基础必备）

使用表格格式，包含：术语、英文全称、通俗解释

```markdown
## 术语表（小白必读）

在开始之前，先了解这些关键术语：

| 术语 | 英文全称 | 通俗解释 |
|------|----------|----------|
| **CLI** | Command Line Interface | 命令行界面，通过打字操作电脑 |
| **API** | Application Programming Interface | 软件之间"对话"的方式 |
| **Token** | - | AI处理文字的计费单位，约等于0.75个英文单词 |
```

**要求**：
- 10-15个核心术语
- 按出现顺序排列

### 2.4 章节结构（根据教程类型选择）

**安装配置类教程**：
```
第一部分：简介与价值（Why - 为什么要学这个）
第二部分：系统要求检查（前置条件）
第三部分：核心安装步骤（重点，每步都要验证）
第四部分：账号/认证配置
第五部分：Hello World验证（确认安装成功）
第六部分：IDE集成配置（可选）

总结与检查清单
附录：命令速查表

```

第四部分：高级用法（进阶技巧）
第五部分：最佳实践（推荐的使用方式）
第六部分：常见问题
总结与速查表
```

## 三、写作格式规范

### 3.1 步骤编写格式（必须遵循）

```markdown
#### 步骤X：[操作名称]

**这一步要做什么**：[一句话说明目的]

**操作方法**：

1. 第一步具体操作
   > 💡 **说明**：解释这个操作的作用

2. 第二步具体操作
   > ⚠️ **注意**：重要提醒

**验证是否成功**：
```bash
# 验证命令
命令
# 预期输出：xxx
```

**如果出错怎么办**：

**错误1**：`错误信息`
**原因**：原因分析
**解决方案**：解决步骤
```

### 3.2 代码块格式

**要求**：
1. ✅ 必须标注语言（小写）：`bash`, `python`, `json`, `powershell`
2. ✅ 每个命令前加注释说明
3. ✅ 预期输出用 `# 预期输出：xxx` 标注
4. ✅ 跨平台命令分开写

**示例**：

**Windows系统（PowerShell）：**
```powershell
# 查看Node.js版本
node --version
# 预期输出：v22.x.x
```

**macOS/Linux系统：**
```bash
# 查看Node.js版本
node --version
# 预期输出：v22.x.x
```

### 3.3 视觉引导标记

```markdown
> 💡 **提示**：有用的小技巧

> ⚠️ **警告**：重要提醒，不注意可能出错

> ❌ **错误做法**：常见错误示例

> ✅ **正确做法**：推荐的方式

> 📌 **信息来源**：[官方文档](URL)

> 🎯 **关键点**：核心要点，必须理解
```

### 3.4 对比表格

用表格展示对比信息：

```markdown
| 对比项 | 选项A | 选项B | 选项C |
|--------|-------|-------|-------|
| 特点1 | ✅ 支持 | ❌ 不支持 | ⚠️ 部分支持 |
| 特点2 | xxx | xxx | xxx |
```

## 五、应用流程

当用户说"直播教程"时，执行以下流程：

### 步骤1：资料验证（WebSearch）
```
1. 搜索软件最新版本
2. 搜索官方安装指南
3. 搜索常见问题和解决方案
4. 记录所有信息来源
```

### 步骤2：内容分析
```
1. 提取所有专业术语
2. 识别需要解释的概念
3. 标记需要验证的步骤
4. 找出可能出错的地方
```

### 步骤3：结构重组
```
1. 按章节结构重组内容
2. 添加术语表
3. 为每个步骤添加解释和验证
4. 添加错误处理
```

### 步骤4：格式化输出
```
1. 应用标准Markdown格式
2. 添加视觉引导标记
3. 分开跨平台命令
4. 添加信息来源标注
```

### 步骤5：质量检查
```
1. 逐项核对检查清单
2. 确认所有验证方法有效
3. 确认小白能理解每个步骤
```
