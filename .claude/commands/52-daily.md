# 📅 每日热点扫描+自动写作系统

**每天一条命令，自动完成内容生产！**

输入：`/daily`
输出：今日热点报告 + 自动生成1-3篇爆款文章

---

## ⚡ 执行流程（完全自动）

### 阶段1：热点扫描（自动执行）

```
自动调用 /hotspot 命令完成热点扫描
```

输出：今日AI热点报告，包含所有热点的爆款评分

---

### 阶段2：自动筛选值得写的热点（自动执行）

**筛选规则**：
```
评分 >= 20分：🔥🔥🔥 必写！立即生成文章
评分 15-19分：🔥🔥 值得写，列入待写清单
评分 10-14分：🔥 可写，但优先级低
评分 < 10分：❌ 跳过
```

**数量控制**：
- 每日最多自动生成3篇文章
- 优先写评分最高的热点
- 如果没有>=15分的热点，提示"今天没有值得写的热点"

---

### 阶段3：批量生成文章（自动执行）

对每个值得写的热点，自动调用 `/auto-baokuan` 命令：

```
FOR each 热点 in 值得写的热点列表:
    /auto-baokuan [热点关键词]
    等待文章生成完成
    记录结果
END FOR
```

## ⚙️ 配置选项

### 自动生成数量

```yaml
# 在 .claude/config/automation.yaml 中配置
daily_scan:
  max_articles_per_day: 3      # 每天最多生成文章数
  min_score_auto_write: 20     # 自动写作的最低分数
  min_score_recommend: 15      # 推荐写作的最低分数
```

### 发布时间建议

```yaml
# 最佳发布时间
publish_times:
  morning: "08:00"    # 早高峰
  noon: "12:00"       # 午休
  evening: "20:00"    # 晚高峰

# 文章类型对应发布时间
type_publish_time:
  测评类: "08:00"     # 早上发，抢时效
  降价类: "12:00"     # 中午发，刺激购买
  经验故事类: "20:00" # 晚上发，适合深度阅读
```

## ⚠️ 注意事项

1. **网络要求**：需要稳定的网络连接，扫描热点需要访问多个数据源
2. **API额度**：注意MCP服务的API调用限制（brave_web_search 2000次/月）
3. **质量优先**：宁可少发也不要发低质量文章
4. **人工审核**：虽然全自动，但发布前建议快速过一遍
