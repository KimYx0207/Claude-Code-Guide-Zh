# 📝 文章翻新工具

**功能**：别人写过的技术，老金来写得更好，而且从不同角度讲

**核心理念**：
> "同样的技术，我要做得更好；同样的内容，我要换个角度说"

**翻新 = 改进提升 + 角度创新**

**两大核心**：

**1. 做得更好（改进提升）**
- ✅ 优化技术方案（精简配置、提炼核心）
- ✅ 补充踩坑经验（别人没说的坑）
- ✅ 提供更好方法（实测优化后的）
- ✅ 数据更详细（具体场景、具体数字）

**2. 不同角度（切入创新）**
- ✅ 别人讲"是什么"→老金讲"为什么"
- ✅ 别人讲"怎么做"→老金讲"怎么做更好"
- ✅ 别人讲"配置步骤"→老金讲"配置原理"
- ✅ 别人讲"功能介绍"→老金讲"实际体验"

**核心原则**：
- ✅ 直接修改原文件（使用Edit工具）
- ✅ 全文深度改写（不是微调，是重写）
- ✅ 技术信息精准保留
- ✅ 表达方式完全改变

## 翻新的具体做法

### 🎯 角度1：从"教配置"变成"教优化"

**原文角度**：教你怎么配置
```
标题：配置Claude Code + Codex MCP协作
内容：第一步、第二步、第三步...
```

**翻新角度**：教你怎么配得更好
```
标题：这个配法比别人少100行，效果还更好
内容：
- 别人的配法有什么问题
- 老金优化后的配法
- 为什么这样配更好
- 实测对比数据
```

### 🎯 角度2：从"功能介绍"变成"实际体验"

**原文角度**：介绍功能
```
这个工具能做什么、有什么特点、支持什么功能
```

**翻新角度**：讲实际使用
```
老金实际用了两周，发现这几个问题：
1. 哪些场景好用
2. 哪些场景不适合
3. 踩了哪些坑
4. 怎么避坑
```

### 🎯 角度3：从"配置步骤"变成"原理解释"

**原文角度**：告诉你步骤
```
第一步：npm install
第二步：配置MCP
第三步：配置CLAUDE.md
```

**翻新角度**：告诉你为什么
```
为什么要这样配？
- MCP的作用是什么
- CLAUDE.md配置的原理
- 为什么Codex要优先
- 这样配的好处在哪
```

### 🎯 角度4：从"全面介绍"变成"聚焦痛点"

**原文角度**：面面俱到
```
介绍所有功能、所有配置、所有用法
```

**翻新角度**：解决核心痛点
```
只讲最痛的那个点：
- 上下文不互通怎么办
- 配置太复杂怎么简化
- 效率怎么真正提升
```

## 工作流程（7步严格执行）

### 步骤1：检查并修正文件名

**文件名格式要求**:
```
YYYY-MM-DD_标题内容_老金风格.md
```

**检查规则**:
1. 文件名必须以日期开头(格式:YYYY-MM-DD)
2. 必须有下划线分隔
3. 必须以"_老金风格.md"结尾

**如果文件名不符合规范**:
```bash
# ⚠️ 重要：必须使用今天的实际日期，不要硬编码示例日期
# 获取今天日期：date +%Y-%m-%d 或直接用环境变量中的Today's date
# 示例：如果今天是2025-12-02
mv "原文件名.md" "2025-12-02_新标题_老金风格.md"
```

**日期选择原则**:
- ⚠️ **必须使用今天的真实日期**（从环境信息 Today's date 获取）
- **不要照抄示例中的日期**
- 标题从文章第一行提取
- 去掉标题中的"#"和空格

**示例**（假设今天是2025-12-02）:
```
❌ claude-code-codex-collaboration.md
✅ 2025-12-02_上周配了一周MCP_Claude和Codex终于自动配合了_老金风格.md

❌ test-article.md
✅ 2025-12-02_这个配置让效率翻3倍_老金风格.md
```
