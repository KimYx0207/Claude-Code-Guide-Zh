### ✅ 真实性验证（V6.0.1升级，极其重要！）

**问题：**
- 时间线抽象（"测试了两天"） → 读者不信
- 案例空泛（"对比了一下"） → 缺乏说服力
- 全是正面描述 → 太像广告
- 语言太书面 → 不像真人写的

**必须做到：**
```
✅ 真实时间线（必须具体）：
   ❌ 禁止："测试了两天"、"最近"、"前段时间"
   ✅ 必须："周五凌晨"、"这个周末"、"昨晚11点"、"今早起来"

✅ 真实案例（必须具体）：
   ❌ 禁止："某个论文"、"一篇文章"、"对比了一下其他工具"
   ✅ 必须：具体论文标题、具体文章名、具体产品名（至少3个）

✅ 踩坑经历（必须包含）：
   ❌ 禁止：全是正面描述，没有问题
   ✅ 必须：至少1-2个真实的坑（工具的bug、功能限制、使用障碍）

✅ 俚语口语化（必须使用）：
   ❌ 禁止："测试"、"发现"、"非常好"、"非常强"
   ✅ 必须："玩"、"扔进去"、"牛逼"、"骚"、"拉"、"吃灰"

✅ 逗号断句（增加频率）：
   ❌ 禁止：句号太多，"它不做PPT，专门给PPT配图。"
   ✅ 必须：逗号断句，"对，是一键生成PPT。"、"所以啊，这就是..."
```

**真实性验证对比示例：**

❌ **不真实的写法（AI腔）**：
```
测试了两天，发现这个工具很强大。对比了一下其他工具，
它的效果非常好。使用起来非常方便，值得推荐。
```

✅ **真实的写法（人味儿）**：
```
周五凌晨通宵玩到现在，说实话，有点慌。

不是因为这玩意儿复杂，而是太tm简单了。

昨晚把《why-language-models-hallucinate》这篇论文扔进去，
黏土风格的PPT，十几分钟就出来了。

但也有坑，中文字会糊，这块比英文拉一些。
```

### 标题优化技巧

**技巧1：A/B测试**

同一篇文章准备3-5个标题：
```
A. 直给型：Claude Code完整使用指南
B. 数字型：Claude Code让我效率提升300%
C. 悬念型：用了这个AI工具，我再也不加班了
D. 痛点型：写代码写到吐？试试这个AI助手
E. 争议型：AI写代码会抢程序员饭碗吗？老金实测
```

根据数据选最优。

**技巧2：关键词前置**

❌ 不好：`一款让我惊艳的AI写作工具推荐`
✅ 更好：`AI写作工具推荐：这款让我惊艳了`

**技巧3：制造落差**

对比要有冲击：从[小]到[大]，从[慢]到[快]，从[难]到[易]

❌ 没冲击：`用AI后效率提升了`
✅ 有冲击：`用AI后，3小时的工作10分钟搞定`

**技巧4：避免标题党**

❌ 过度夸张：`震惊！AI完全颠覆人类工作方式！`
✅ 克制但有吸引力：`AI如何改变工作方式？老金用了一个月的体会`
