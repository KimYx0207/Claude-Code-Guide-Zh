# Markdown格式规范清单（V7.2.1）

**版本**：V7.2.1 | **更新日期**：2025-12-10

**用途**：写作后格式自检、AI写作时的强制约束

---

## 一、章节分隔规范（最高优先级）

### ✅ 正确做法
```markdown
正文内容最后一行

## 下一个标题
紧贴标题的正文
```

### ❌ 错误做法
```markdown
正文内容最后一行

---
## 下一个标题
```

**规则说明**：
- **不要使用`---`分隔线**（会导致渲染空白过大）
- **直接用`##`标题分隔章节**，标题本身就有分隔效果
- 正文结束后空一行，然后直接写`##`标题

---

## 二、标题格式规范

### 2.1 标题后不空行
```markdown
## 这是标题
紧贴标题的正文内容
```

### ❌ 错误做法
```markdown
## 这是标题

正文内容（标题后有空行）
```

### 2.2 标题简洁性
- ❌ **禁止括号说明**："## Artifacts系统（重点功能）"
- ❌ **禁止"XX有多牛"**："## Claude有多牛"
- ❌ **禁止"XX（推荐）"**："## 方案一（推荐）"
- ✅ **简洁有力**："## Artifacts系统"、"## 方案一：Tun模式"

---

## 三、代码块格式规范

### 3.1 代码块前不留空行
```markdown
正文内容紧贴代码块：
` ``markdown
代码内容
` ``
```

### ❌ 错误做法
```markdown
正文内容

` ``markdown
代码内容
` ``
```

### 3.2 代码块类型选择
| 包含`#`注释？ | 使用类型 |
|--------------|---------|
| 是 | ` ```markdown ` |
| 否（纯命令） | ` ```bash ` |

### 3.3 代码块内禁止
- ❌ 不要用`**加粗**`标记（飞书不渲染）
- ✅ 代码块内容缩进一致（用4个空格，不用Tab）

---

## 四、序号格式规范

### ✅ 正确做法
```markdown
1、第一步
2、第二步
3、第三步
```

### ❌ 错误做法
```markdown
1. 第一步      # markdown列表格式
- 第一步       # bullet列表
### 第一步     # 标题格式
```

---

## 五、表格格式规范（完全禁用）

### ❌ 禁止使用Markdown表格
原因：飞书、公众号、很多平台渲染不好

```markdown
| 名称 | 说明 |
|------|------|
| 项目1 | 描述1 |
```

### ✅ 替代方案1：列表格式
```markdown
**项目1**：描述1
**项目2**：描述2
```

### ✅ 替代方案2：分段格式
```markdown
**项目1**

描述1的详细内容

**项目2**

描述2的详细内容
```

## 六、加粗格式规范

### ✅ 正确用法
- **独立成行加粗**：`**整句话都加粗**`（必须单独一行）
- **强调加粗**：关键句，如`**关键是，免费的。**`

### ❌ 错误用法
- **句子中间加粗**：`这是**部分加粗**的句子`（飞书不渲染）
- **代码块内加粗**：代码块内的`**`标记无效
- **过度加粗**：全文不超过5-6处

---

## 七、引用格式规范

### ✅ 正确做法
```markdown
正文内容

> 这是引用内容

引用后的正文
```

**规则**：引用前后各空一行

---

## 八、换行规范

### 8.1 一句一行
```markdown
这是第一句话。
这是第二句话。
这是第三句话。
```

### 8.2 冒号后换行（但不空行）
```markdown
OpenAI这次给了两个选择：
一个叫Instant即时版，一个叫Thinking思考版。
```

### 8.3 引号单独成段
```markdown
正文内容

> "这是一段引用内容"

引用后的正文
```

**重要**：引号内的句号、问号、冒号**不换行**！引号内容保持完整。

### 8.4 段落判断标准

**同一段落（句子间不空行）**：
- 逻辑紧密的并列
- 问答关系
- 冒号引出说明
- 主题+例子
- 同主题多方面描述
- 对比关系

**独立段落（前后空行）**：
- 独立观点/总结
- 话题转折
- 标题

## 九、图片格式规范

### ✅ 正确做法
```markdown
正文内容

![](图片URL)

图片后的正文
```

### 规则说明
- alt留空（方括号内不写字）
- 无图片说明/署名
- 上下各空一行

## 十、格式自检清单

### 写作完成后必须检查：

**分隔符检查**
- [ ] 没有使用`---`分隔线
- [ ] 章节间只用`##`标题分隔

**标题检查**
- [ ] 标题后没有空行
- [ ] 标题简洁，无括号说明

**代码块检查**
- [ ] 代码块前没有空行
- [ ] 有`#`注释的用` ```markdown `
- [ ] 代码块内没有`**加粗**`标记

**序号检查**
- [ ] 使用"1、2、3、"格式
- [ ] 没有使用"1."或"-"列表格式

**表格检查**
- [ ] 没有使用Markdown表格
- [ ] 已用列表或分段格式替代

**加粗检查**
- [ ] 加粗都是独立成行
- [ ] 全文加粗不超过5-6处

## 快速参考卡片

| 场景 | 正确做法 | 错误做法 |
|------|---------|---------|
| 章节分隔 | `\n\n## 标题` | `\n\n---\n## 标题` |
| 标题后 | `## 标题\n正文` | `## 标题\n\n正文` |
| 代码块前 | `正文：\n```markdown` | `正文：\n\n```markdown` |
| 序号 | `1、2、3、` | `1. 2. 3.` 或 `- - -` |
| 表格 | 用列表替代 | Markdown表格 |
| 加粗 | `**整行加粗**` | `部分**加粗**部分` |
