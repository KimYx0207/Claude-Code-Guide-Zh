# 测评类文章模板（爆款公式1）

**预期阅读**：5000-15000
**核心要求**：新品发布12小时内发文！

---

## 标题公式（5选1）

```
1. 时效+排名型：昨晚/今天 + 上线/发布 + 就拿第一 + 核心卖点
   示例：昨晚上线就拿第一，100万token一口气读完

2. 能力飞跃型：产品名 + 核心能力 + 数字倍数 + 旧版对比
   示例：Cursor记忆扩4倍，V1的坑终于填了

3. 版本冲击型：产品名 + V2/2.0 + 情绪词 + 核心升级点
   示例：Cursor V2来了，这升级有点狠

4. 数据对比型：产品名 + 具体数据提升 + 对比
   示例：32K扩到128K，Claude的记忆力终于够用了

5. 意外发现型：我/老金 + 动作 + 后发现 + 意外结果
   示例：深度体验Claude一周后，我卸载了5个插件
```

**标题检查**：
- [ ] 长度15-25字？
- [ ] 有数字冲击？
- [ ] 有情绪词？
- [ ] 关键词前置？

---

## 文章结构（4段式）

### 第1段：制造悬念（3-5行）

**必须包含**：
- ✅ 具体时间（周五凌晨/昨晚11点/今早起来）
- ✅ 情绪冲击（慌了/吓一跳/懵了/有点狠）
- ✅ 悬念铺垫（不直接说更新内容）

**模板**：
```
[具体时间]刷[信息源]，看到[产品名]的[更新/发布]，
我整个人有点[情绪词]。

不是因为[常规原因]，而是[真正原因]，
有点tm[情绪词]。
```

**示例**：
```
周五凌晨刷GitHub，看到Cursor V2的更新日志，
我整个人有点慌。

不是因为bug，而是这升级的力度，
有点tm狠。
```

❌ **禁止**：
- 直接说"XX发布V2版本"（太干）
- 没有时间具体化
- 没有情绪词

---

### 第2段：新旧对比（10-15行）

**必须包含**：
- ✅ 旧版本的3-5个真实使用场景
- ✅ 旧版本的痛点/限制（至少2个）
- ✅ 过渡句引出新版本

**模板**：
```
[旧版本]，我用了[时间]，真实场景是：

- [场景1]，[具体限制1]
- [场景2]，[具体限制2]
- [场景3]，[具体限制3]

核心痛点就一个：**[痛点总结]**。

但[新版本]把这问题给解决了。
```

**对比表格模板**：
```markdown
| 场景 | 旧版本 | 新版本 |
|------|--------|--------|
| 场景1 | 具体限制1 | 具体改进1 |
| 场景2 | 具体限制2 | 具体改进2 |
| 场景3 | 具体限制3 | 具体改进3 |
```

---

### 第3段：核心更新详解（30-50行，最重要）

**必须包含**：
- ✅ 3-5个核心更新点（使用1、2、3、编号）
- ✅ 每个更新点配实战案例
- ✅ 每个更新点有数据对比
- ✅ 每个更新点说明实际影响

**每个更新点的结构**：
```markdown
### 1、[更新点名称] - [一句话价值]

**旧版的坑**：
[具体描述旧版痛点，3-5行]

**新版的改进**：
[具体描述新版能力，5-8行]

**实战案例**：
[真实使用场景，10-15行]
- 场景描述
- 操作过程
- 结果展示（必须有数据/截图/代码）

**数据对比**：
- 旧版：[具体数据]
- 新版：[具体数据]
- 提升：[百分比/倍数]
```

---

### 第4段：真实踩坑 + 避坑指南（10-15行）

**必须包含**：
- ✅ 至少2个真实的坑（不能全是好话！）
- ✅ 每个坑的具体表现
- ✅ 避坑方法/workaround

**模板**：
```markdown
**真实的坑（必须说）**：

1、**[坑1名称]**
   [具体表现，2-3行]

   **避坑方法**：[解决方案]

2、**[坑2名称]**
   [具体表现，2-3行]

   **影响**：[影响程度评估]
```

**示例**：
```markdown
**真实的坑（必须说）**：

1、**中文提示词还是拉**
   昨晚用中文让它写个工具函数，
   生成的代码注释全是英文，
   切换成英文提示词就正常了。

   **避坑方法**：复杂需求用英文描述

2、**响应速度慢了**
   V2的响应时间比V1慢20-30%，
   可能是模型更大了。

   **影响**：对话体验稍微卡一点，能接受
```

❌ **禁止**：
- 全是正面描述（太像广告）
- 抽象说"有些小问题"
- 没有解决方法

---

## 发文前检查清单

### 时效性检查
- [ ] 新品发布后12小时内？
- [ ] 不是翻译/解读二手信息？

### 真实性检查
- [ ] 时间线具体（周五凌晨/昨晚，不是"最近"）？
- [ ] 案例具体（产品名/数据，不是"某个工具"）？
- [ ] 有踩坑经历（至少2个真实问题）？
- [ ] 使用俚语（玩/扔/牛逼，不是"测试/发现"）？

### 内容检查
- [ ] 有3-5个核心更新点？
- [ ] 每个点有实战案例？
- [ ] 有数据对比（新旧版本）？
- [ ] 有社区反馈引用（Reddit/HN至少5条）？

### 标题检查
- [ ] 使用了5大标题公式之一？
- [ ] 有数字冲击？
- [ ] 有情绪词？
- [ ] 长度15-25字？

---

**记住**：测评类文章的价值不是转述官方信息，而是**帮读者搞清楚：升不升级、怎么用、有什么坑**。
