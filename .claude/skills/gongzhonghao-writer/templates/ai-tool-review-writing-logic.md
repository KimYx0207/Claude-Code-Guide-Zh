# AI工具测评写作逻辑（方法论）

> 基于《Google发布Nano Banana Pro》提炼
> **这是思维方式，不是内容模板**

---

## 底层逻辑：让读者从"这和我有啥关系"到"我要转发"

```
标题吸引 → 痛点共鸣 → 解决方案 → 案例验证 → 对比决策 → 行动引导
   ↓          ↓          ↓          ↓          ↓          ↓
 3秒判断    "对，就是我"  "这个牛逼"  "真的能用"  "我该选哪个"  "现在就试"
```

---

## 一、标题设计逻辑

### 核心原则：3秒内让读者判断"这篇文章和我有关"

### 5种标题公式

| 公式 | 结构 | 心理触发 | 示例 |
|------|------|----------|------|
| **直接利益型** | [工具]终于解决了[痛点] | 立即获得感 | "AI生图终于能写清楚中文字了" |
| **问题否定型** | [痛点]不再是[痛点] | 痛点消失的爽感 | "AI生图写中文不再是乱码" |
| **场景替代型** | 做[场景]再也不用[旧方案]了 | 效率提升想象 | "做海报再也不用PS了" |
| **对比冲突型** | [错误案例]不再变成[错误结果] | 具体场景共鸣 | "老金面馆不再变成老玉而官" |
| **人群刺激型** | [人群]狂喜：终于能[诉求]了 | 身份认同感 | "电商人狂喜：AI终于能写对中文" |

### 设计关键
- ❌ 不要：纯信息型（"Google发布新版本"）
- ✅ 要：场景+痛点+解决方案

---

## 二、开篇公式：3句话定律

```
第1句：信息层 - 今天发生了什么（工具名+核心升级罗列）
第2句：价值层 - 对目标读者最重要的是什么【粗体强调】
第3句：过渡层 - 我先说说背景（建立对话感）
```

### 心理设计
- 第1句：让读者知道"有新东西了"
- 第2句：让读者产生"这和我有关"
- 第3句：让读者愿意"听你讲背景"

### 关键原则
- 不铺垫，不绕弯，直接说最重要的点
- 用"老金我先说说"建立聊天感，降低阅读门槛

---

## 三、痛点共鸣公式：让读者说"对，就是我！"

### 结构设计

```
背景铺垫（工具是什么 + 优点 + 但有缺点）
       ↓
痛点引入："你是不是也遇到过这种情况："
       ↓
3个具体场景（从读者生活/工作中找）
       ↓
总结痛点："这就是XX最大的痛点"【粗体】
```

### 场景选择原则

| 原则 | 说明 | 反例 |
|------|------|------|
| **具体不抽象** | "老金面馆变成老玉而官" | ❌"中文渲染不好" |
| **常见不罕见** | 做产品海报、餐饮招牌、春节海报 | ❌"给NASA设计Logo" |
| **有画面感** | 读者脑海里能想象出场景 | ❌"文字识别率低" |

### 心理设计
- 3个场景覆盖不同人群（电商、餐饮、节日）
- 用"你是不是也"建立共情
- 用错误案例引发"我也遇到过"的回忆

## 四、解决方案展示逻辑

### 结构设计

```
宣布新版本解决了痛点
       ↓
具体改进点（用数据/对比说话）
       ↓
横向对比竞品（衬托价值）
       ↓
适用人群列举（让读者对号入座）
       ↓
总结："这次升级是真正的质变"【粗体】
```

### 关键原则

1. **数据说话**
   - ❌ "渲染质量很好"
   - ✅ "简体繁体都能写对，笔画完整"

2. **对比衬托**
   - 竞品A、B、C怎么样（客观说明，不贬低）
   - 新工具达到了什么水平

3. **人群对号入座**
   ```
   如果你是：
   - 做电商的：具体好处
   - 做餐饮的：具体好处
   - 做设计的：具体好处
   ```

## 五、案例设计逻辑（核心方法论）

### 黄金公式：娱乐性 + 实用性 + 传播性

| 维度 | 设计方法 | 检验标准 |
|------|----------|----------|
| **娱乐性** | 梗、跨次元、萌宠、IP | 读者会笑/会惊讶 |
| **实用性** | 真实工作场景、刚需 | 读者能马上用 |
| **传播性** | 情感共鸣、身份认同 | 读者想转发/收藏 |

### 案例结构（固定模板）

```
### 案例X：具体场景名称（目标人群反应）

想做一个XX，要求XX。

**提示词**：
> 中文提示词（必须包含尺寸比例）

**[此处放图：图片描述]**

**为啥这个有传播性**：一句话说明（人群+痛点+情感）
```

### 案例数量分配

| 功能类型 | 案例数 | 案例类型分配 |
|----------|--------|--------------|
| 差异化杀手锏 | 3个 | 创业/职场类 + 生活/城市类 + 娱乐/总结类 |
| 核心功能 | 2-3个 | 1个偏实用 + 1个偏娱乐 + 1个中间 |
| 次要功能 | 不展开 | 简单列举即可 |

### 案例选择决策树

```
                  新功能
                    ↓
            是否有差异化？
           ↙            ↘
        是（杀手锏）      否
          ↓               ↓
      3个案例          2个案例
          ↓               ↓
    创业/生活/娱乐    实用+娱乐
```

### 提示词设计原则

1. **必须中文**（老金人设：不会代码，不用英文）
2. **必须包含尺寸比例**
   - 9:16 - 竖版（微信红包、手机壁纸）
   - 16:9 - 横版（桌面壁纸、PPT）
   - 1:1 - 方形（社交媒体发帖）
   - 3:4 - 书籍封面
   - 2:3 - 电影海报
   - 21:9 - 宽幅横幅（店招）
3. **可执行性**：读者复制就能用，不要有占位符

### 传播性设计方法

**公式**：目标人群 + 刚需场景 + 情感触发

| 人群 | 刚需场景 | 情感触发 | 示例 |
|------|----------|----------|------|
| 打工人 | 房价、职场焦虑 | "这说的就是我" | 北京房价15年变化 |
| 创业者 | 成功案例 | "我也想这样" | 小米14年创业史 |
| 铲屎官 | 宠物娱乐 | "我家主子也是" | 宠物三变装 |
| 家庭用户 | 全家福 | "真实需求" | 一家五口年度全家福 |
| 游戏玩家 | IP联动 | "梦想成真" | 游戏角色大乱斗 |

## 六、差异化功能展示逻辑

### 核心原则：这个功能必须让读者产生"我靠，牛逼！"

### 结构设计

```
引出独特性："这个功能必须单独说"
       ↓
强调差异："其他工具都做不到"【粗体】
       ↓
功能说明："具体能干什么"
       ↓
案例验证（3个案例，覆盖不同人群）
       ↓
价值总结："这是XX的独家优势"
```

### 差异化判断标准

| 判断维度 | 是差异化 | 不是差异化 |
|----------|----------|------------|
| 竞品对比 | ✅ 只有这个工具能做到 | ❌ 竞品也能做，只是差一点 |
| 用户感知 | ✅ 用户会说"我靠，这个牛" | ❌ 用户说"这个不错" |
| 使用门槛 | ✅ 降低了原本不可能的门槛 | ❌ 只是优化了体验 |

---

## 七、横向对比逻辑：帮用户做选择，不是帮工具打广告

### 核心原则：实用主义，不偏袒

### 结构设计

```
直说最强的点
       ↓
说出大问题（比如需要VPN）
       ↓
国产工具的优势（客观说明）
       ↓
选择建议（4-5个if-then判断）
       ↓
总结："别追求最强，追求好用"【粗体】
```

### 对比话术设计

| 对比维度 | 话术模板 | 心理设计 |
|----------|----------|----------|
| 最强的点 | "老金我直说了：XX最强的三个点" | 建立信任：我不瞎吹 |
| 大问题 | "但有个大问题：XX" | 平衡：我也说缺点 |
| 国产优势 | "国产工具的优势：ABC" | 客观：不贬低任何一方 |
| 选择建议 | "你是XX情况→用XX工具" | 实用：帮你做决策 |

### 选择建议公式

```
if 你是[用户画像] and 你的需求是[场景]
then 用[工具A]，因为[理由]

if 你是[用户画像] and 你的需求是[场景]
then 用[工具B]，因为[理由]
```

### 总结金句设计

**类比法**：
```
就像[生活场景]，[A工具]确实[优点]，
但你在[特定条件]，[B工具]反而更实用
```

示例：
> 就像开车，法拉利确实快，但你在北京早高峰堵车，比亚迪反而更实用

## 八、行动引导逻辑：降低门槛

### 双管齐下

**1. 即时行动**：现在就能用
```
- XX今天上线
- 在这些地方能用：
  - 个人用户：怎么用
  - 开发者：怎么用
```

**2. 学习资源**：别瞎编，直接抄
```
- 官方指南（1-2个）
- 社区案例（4-5个）
- 每个资源一句话说明价值
```

### 心理设计
- "现在就能用"→降低行动门槛
- "别瞎编，直接抄"→降低学习成本

## 九、总结升华逻辑：输出观点，不是总结内容

### 核心原则：从工具评测上升到行业洞察

### 结构设计

```
核心观点（一句话）【粗体】
       ↓
客观分析（XX确实强，但YY是门槛）
       ↓
平衡视角（国产工具的价值）
       ↓
类比说明（生活化场景类比）
       ↓
行业洞察（工具竞争的本质是什么）
```

### 观点公式

**公式1：本质论**
```
[领域]选择本质上是选择"[维度A]"，不是选择"[维度B]"
```
示例："工具选择本质上是选择'适合自己'，不是选择'最强'"

**公式2：实用论**
```
对[人群]来说，[价值A]比[价值B]更重要
```
示例："对国内用户来说，能稳定使用比技术参数更重要"

**公式3：竞争论**
```
[行业]的竞争，最终拼的不是[表面指标]，是[本质价值]
```
示例："AI工具的竞争，最终拼的不是参数，是用户能不能真正用起来"

## 十、语言风格原则（老金人设）

### 人设定位
- 产品经理视角，不会代码
- 关心"能不能用"，不关心"怎么实现的"
- 实用主义者，不追求最强，追求好用

### 语言特征

| 维度 | 要求 | 反例 |
|------|------|------|
| 术语翻译 | 技术术语翻译成人话 | ❌"API调用" ✅"开发者能接入" |
| 序号格式 | 用顿号：1、2、3、 | ❌ 用句点：1. 2. 3. |
| 数据优先 | 能用数字不用形容词 | ❌"很快" ✅"快2-3倍" |
| 案例真实 | 有场景感，不虚构 | ❌"某公司" ✅"小米" |
| 对话感 | "老金我"、"你是不是也" | ❌ 书面语 |

## 使用方法

### 步骤1：确定核心差异化（1-3个）
问自己：这个工具最牛逼的点是什么？其他工具做不到的是什么？

### 步骤2：设计痛点场景（3个）
问自己：用户在什么场景下会痛苦？能不能用一句话描述错误案例？

### 步骤3：设计案例（每个功能2-3个）
问自己：
- 这个案例有娱乐性吗？（梗、跨次元、萌宠）
- 这个案例有实用性吗？（真实工作场景）
- 这个案例有传播性吗？（目标人群会转发吗？）

### 步骤4：写提示词
确保：中文 + 尺寸比例 + 可执行

### 步骤5：横向对比
客观说：最强点 + 大问题 + 国产优势 + 选择建议

### 步骤6：总结升华
输出观点，不是总结内容。用类比法让观点更好理解。
