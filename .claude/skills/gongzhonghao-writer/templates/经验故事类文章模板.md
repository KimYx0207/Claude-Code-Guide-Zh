# 经验故事类文章模板（爆款公式3）

**预期阅读**：4000-7000
**核心要求**：真实经历、具体数据、踩坑教训、实用建议

---

## 标题公式（5选1）

```
1. 时间维度型：用了X年/做了X个项目 + 才知道/才发现 + 价值点
   示例：用AI写代码2年，才知道这个窍门
   示例：做了100个项目，踩过的5个大坑

2. 踩坑避雷型：用/做 + XX + 千万别 + 错误做法
   示例：用ChatGPT写公众号千万别直接复制粘贴
   示例：学AI千万别从编程开始

3. 意外发现型：我/老金 + 动作 + 后，发现 + 意外结果
   示例：我用AI写了30篇文章后，发现了这个秘密
   示例：深度体验Claude Code一周后，我卸载了5个插件

4. 反常识型：反常识观点 + 但大多数人不知道
   示例：AI不会抢你的工作，但不用AI的人会
   示例：学AI不用懂编程，但要懂这3个思维

5. 数字对比型：工具/方法 + 让我 + 旧状态 → 新状态
   示例：Claude Code让我从一天200行到2000行
   示例：用了这个方法，写作时间从3小时缩到30分钟
```

**标题检查**：
- [ ] 有时间/数量维度？
- [ ] 有具体数字？
- [ ] 有价值承诺？
- [ ] 长度15-25字？

---

## 文章结构（4段式）

### 第1段：背景故事（5-8行）

**必须包含**：
- ✅ 具体时间线（X年前/X个月前）
- ✅ 当时的状态/痛点
- ✅ 为什么开始做这件事
- ✅ 情绪共鸣点

**模板**：
```
[时间]前，我还是个[状态描述]。

那时候[痛点描述]，
[具体困难1]，[具体困难2]。

直到[转折点]，一切开始变了。
```

**示例**：
```
2年前，我还是个每天加班到11点的码农。

那时候写代码全靠硬刚，
一个功能磨3天，改bug改到怀疑人生。

直到试了Claude Code，一切开始变了。
```

❌ **禁止**：
- 时间线抽象（"很久以前"、"之前"）
- 没有情绪共鸣
- 直接说教

---

### 第2段：踩坑经历（15-20行，最重要！）

**必须包含**：
- ✅ 3-5个真实踩过的坑
- ✅ 每个坑的具体表现
- ✅ 当时的错误做法
- ✅ 造成的后果

**模板**：
```
**这[X]年，我踩过的坑**：

### 坑1：[坑名称]

[当时的做法，2-3行]

结果呢？[后果描述]

### 坑2：[坑名称]

[当时的做法，2-3行]

后来才知道，[正确认知]

### 坑3：[坑名称]
...
```

**示例**：
```
**这2年，我踩过的坑**：

### 坑1：把AI当搜索引擎用

刚开始用Claude，我就像用百度一样，
问"Python怎么写循环"这种问题。

结果呢？效率还不如直接Google，浪费了3个月。

### 坑2：不给上下文

直接说"帮我写个函数"，
不告诉它项目背景、代码风格、依赖库。

生成的代码根本没法用，每次都要大改。

### 坑3：不迭代，一次定稿

以为AI一次就能写对，
结果代码质量参差不齐，bug一堆。

后来才知道，要跟AI对话，不断优化。
```

---

### 第3段：解决方案/经验总结（15-20行）

**必须包含**：
- ✅ 3-5个实用建议/方法
- ✅ 每个建议的具体操作
- ✅ 预期效果/数据
- ✅ 可直接复制的模板/提示词（如果有）

**模板**：
```
**踩完坑后，我总结了[X]个方法**：

### 方法1：[方法名称]

**怎么做**：
[具体操作步骤]

**效果**：
[具体数据/案例]

### 方法2：[方法名称]
...
```

**示例**：
```
**踩完坑后，我总结了3个方法**：

### 方法1：给足上下文

**怎么做**：
每次提问前，先告诉AI：
- 项目背景（做什么的）
- 技术栈（Python/React/...）
- 代码风格（命名规范、注释要求）
- 具体需求（输入输出）

**效果**：
代码可用率从30%提升到80%，改动量减少一半。

### 方法2：迭代优化，不要一次定稿

**怎么做**：
1、先让AI出初版
2、指出问题，让它改
3、再指出问题，再改
4、满意为止

**效果**：
虽然对话轮次多了，但最终代码质量高很多。
```

---

### 第4段：总结 + 行动建议（5-8行）

**必须包含**：
- ✅ 核心观点总结（1-2句）
- ✅ 给读者的具体行动建议
- ✅ 情绪共鸣/鼓励

**模板**：
```
**总结一下**：

[核心观点，1-2句]

如果你也[读者状态]，建议：
1、[行动1]
2、[行动2]
3、[行动3]

[鼓励/共鸣的话]
```

**示例**：
```
**总结一下**：

AI工具只是工具，关键是怎么用。
用对了，效率翻倍；用错了，还不如不用。

如果你也刚开始用AI写代码，建议：
1、先花1小时学学提示词技巧
2、从小任务开始，别一上来就搞大项目
3、多迭代，别指望一次搞定

踩坑不可怕，怕的是踩完不总结。
```

## 真实性强化技巧

### 技巧1：具体数字
```
❌ 效率提升了很多
✅ 效率从每天200行提升到800行

❌ 用了很长时间
✅ 用了2年3个月

❌ 做了很多项目
✅ 做了47个项目
```

### 技巧2：具体场景
```
❌ 有一次写代码遇到问题
✅ 上周五重构用户登录模块，遇到一个JWT过期的bug

❌ 用AI帮我写了个功能
✅ 让Claude帮我写了个自动生成周报的Python脚本
```

### 技巧3：情绪真实
```
❌ 这个方法很好用
✅ 第一次试的时候，我整个人都傻了，这也太省事了

❌ 遇到了一些困难
✅ 当时改bug改到凌晨3点，差点把电脑砸了
```

### 技巧4：口语化表达
```
❌ 我发现这个工具非常强大
✅ 这玩意儿，真tm牛逼

❌ 测试了一下效果
✅ 扔进去跑了一下

❌ 非常推荐大家使用
✅ 反正我是回不去了，谁用谁知道
```

## 发文前检查清单

### 真实性检查（最重要！）
- [ ] 有具体时间线（X年/X个月）？
- [ ] 有具体数字（X个项目/X行代码）？
- [ ] 有3个以上真实踩坑经历？
- [ ] 有口语化表达？
- [ ] 情绪真实不做作？

### 实用性检查
- [ ] 有3-5个可操作的方法？
- [ ] 每个方法有具体步骤？
- [ ] 有可直接复制的模板/提示词？

### 结构检查
- [ ] 背景故事引人入胜？
- [ ] 踩坑经历够痛？
- [ ] 解决方案够实用？
- [ ] 结尾有行动号召？

### 标题检查
- [ ] 有时间/数量维度？
- [ ] 有价值承诺？
- [ ] 长度15-25字？

---

**记住**：经验故事类文章的价值是**让读者从你的踩坑中学到东西**，不是炫耀你多厉害。越真实、越具体、越有人味儿，读者越爱看。
