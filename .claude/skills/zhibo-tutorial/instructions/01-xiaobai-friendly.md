# 小白友好写作指南

**核心原则**：假设读者完全不懂技术，每个概念都要解释清楚。

---

## 一、三问法则（每个操作必须回答）

### 1. 这是什么？

**要求**：用生活类比解释技术概念

**示例**：
```markdown
### Node.js

**这是什么？**
Node.js就像一个"翻译官"，让你的电脑能够运行JavaScript代码。
就像你需要安装微信才能聊天一样，需要安装Node.js才能运行Claude Code。
```

**常用类比表**：

| 技术概念 | 生活类比 |
|----------|----------|
| API | 餐厅菜单，告诉你能点什么 |
| Token | 手机流量，用多少算多少钱 |
| CLI | 发短信给电脑，打字交流 |
| 环境变量 | 电脑的"备忘录"，存储重要信息 |
| 包管理器 | 应用商店，下载安装软件 |
| 终端 | 和电脑对话的窗口 |
| 路径 | 文件的"地址" |
| 权限 | 钥匙，决定能打开哪些门 |

### 2. 为什么要做？

**要求**：说明这个操作的目的和价值

**示例**：
```markdown
**为什么要安装？**
Claude Code是用JavaScript写的，没有Node.js就无法运行。
这是必装的基础环境，就像手机没有系统就不能开机一样。
```

### 3. 怎么操作？

**要求**：具体步骤 + 每步说明 + 预期结果

**示例**：
```markdown
**操作步骤：**

1. 打开浏览器，访问 https://nodejs.org/
   > 💡 **说明**：这是Node.js官方网站，安全可靠，不要从其他地方下载

2. 点击绿色的"LTS"按钮下载
   > 💡 **说明**：LTS是"长期支持版"的意思，比"Current"版本更稳定

3. 双击下载的安装包
   > 💡 **说明**：Windows是.msi文件，macOS是.pkg文件

4. 一路点"Next"或"继续"
   > 💡 **说明**：保持默认选项即可，不需要修改任何设置
```

打开**终端**（Terminal，就是一个黑色的窗口，用来输入命令和电脑"对话"）

**示例**：

首先需要设置**环境变量**（电脑的"备忘录"，用来存储一些重要信息，程序运行时会自动读取）

## 三、操作说明标记

### 必用标记

```markdown
> 💡 **说明**：解释这个操作的作用，为什么要这样做

> ⚠️ **注意**：重要提醒，不注意可能出错

> 🎯 **关键**：核心要点，必须理解

> ❌ **常见错误**：小白容易犯的错误

> ✅ **正确做法**：推荐的方式
```

### 使用示例

```markdown
1. 打开终端
   > 💡 **说明**：Windows用户按 `Win+R`，输入 `cmd` 回车
   > ⚠️ **注意**：不是"运行"窗口本身，是打开后出现的黑色窗口

2. 输入命令 `node --version`
   > 💡 **说明**：这个命令是问电脑"Node.js装好了吗？版本是多少？"
   > ❌ **常见错误**：不要输入引号，直接输入 node --version
   > ✅ **正确做法**：输入后按回车键执行
```

## 五、常见小白疑问预答

在可能产生疑问的地方，主动回答：

```markdown
> 🤔 **你可能会问**：LTS和Current有什么区别？
>
> **答**：LTS（Long Term Support）是长期支持版，更稳定，适合大多数用户。
> Current是最新版，有新功能但可能不够稳定。初学者选LTS就对了。
```
