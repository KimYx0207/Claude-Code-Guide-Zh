# 直播教程写作工作流程

**核心理念**：按照标准化流程写教程，确保质量一致。

---

## 一、完整工作流程（5个阶段）

```
阶段1：资料准备 → 阶段2：内容分析 → 阶段3：结构重组 → 阶段4：内容编写 → 阶段5：质量验证
```

---

## 阶段1：资料准备（WebSearch验证）

### 1.1 确定教程主题

**输入**：用户提供的原始资料（备份文件、笔记等）

**输出**：清晰的教程主题定义

**示例**：
```
原始资料：Claude Code安装笔记.txt
教程主题：Claude Code完整安装指南
教程类型：安装配置类
目标读者：零基础小白
```

### 1.2 执行WebSearch验证

**必须验证的信息**：

```markdown
## WebSearch验证清单

□ 软件最新版本号
   搜索：[软件名] latest stable version 2025

□ 官方安装指南
   搜索：[软件名] official installation guide

□ 系统要求
   搜索：[软件名] system requirements

□ 常见错误和解决方案
   搜索：[软件名] common installation errors solutions

□ 中国用户特殊注意事项（如镜像源）
   搜索：[软件名] 国内安装 镜像
```

**输出**：验证信息记录表

```markdown
## 验证信息记录

| 信息项 | 验证内容 | 来源URL | 验证日期 |
|--------|----------|---------|----------|
| Node.js版本 | v22.11.0 LTS | nodejs.org | 2025-12-17 |
| Claude Code版本 | v2.0.71 | npmjs.com | 2025-12-17 |
| ... | ... | ... | ... |
```

---

## 阶段2：内容分析

### 2.1 提取术语清单

**操作**：从原始资料中识别所有专业术语

**输出格式**：
```markdown
## 术语清单

1、Node.js - 需要解释什么是运行时
2、npm - 需要解释什么是包管理器
3、CLI - 需要解释命令行界面
4、API Key - 需要解释什么是密钥
5、环境变量 - 需要解释配置概念
...（至少10个）
```

### 2.2 识别需要验证的步骤

**操作**：标记所有需要验证的操作

**输出格式**：
```markdown
## 需要验证的步骤

1、Node.js安装 → 验证命令：node --version
2、npm配置 → 验证命令：npm config get registry
3、Claude Code安装 → 验证命令：claude --version
4、API Key配置 → 验证命令：echo $ANTHROPIC_API_KEY
...
```

### 2.3 识别可能出错的地方

**操作**：标记常见错误点

**输出格式**：
```markdown
## 错误风险点

1、权限问题 → npm install -g 可能需要管理员权限
2、网络问题 → npm默认源在国外，国内可能超时
3、版本冲突 → Node.js版本过低会导致安装失败
...
```

---

## 阶段3：结构重组

### 3.1 选择章节模板

**安装配置类教程**（如本例）：
```
第一部分：简介与价值
第二部分：系统要求检查
第三部分：核心安装步骤
第四部分：配置与认证
第五部分：Hello World验证
第六部分：常见问题
第七部分：总结与速查表
```

**功能使用类教程**：
```
第一部分：功能概述
第二部分：快速开始
第三部分：核心功能详解
第四部分：高级用法
第五部分：最佳实践
第六部分：常见问题
第七部分：总结与速查表
```

### 3.2 规划每个章节

**章节规划表**：
```markdown
## 章节规划

| 章节 | 主要内容 | 预计篇幅 | 验证点 |
|------|----------|----------|--------|
| 简介 | 什么是Claude Code、能做什么 | 500字 | 无 |
| 系统要求 | 操作系统、硬件、网络 | 300字 | 系统版本检查 |
| 安装Node.js | 下载、安装、验证 | 800字 | node --version |
| 安装Claude Code | npm安装、验证 | 600字 | claude --version |
| ... | ... | ... | ... |
```

---

## 阶段4：内容编写

### 4.1 编写术语表

**输入**：阶段2的术语清单

**输出**：完整的术语表（飞书列表格式）

```markdown
## 术语表（小白必读）

**CLI**
**英文全称**：Command Line Interface
**通俗解释**：命令行界面，通过打字操作电脑
**生活类比**：像发短信一样和电脑"聊天"

**Node.js**
**英文全称**：Node.js
**通俗解释**：JavaScript运行环境
**生活类比**：像播放器，让电脑能运行JS代码

...（10-15个术语）
```

### 4.2 编写每个章节

**每个章节必须包含**：
1、章节标题
2、"这是什么"解释
3、"为什么要做"说明
4、具体操作步骤（每步都有💡说明）
5、验证方法
6、错误处理

**章节编写模板**：
```markdown
## 第X部分：[章节名称]

### X.1 [小节名称]

**这是什么？**
[用生活类比解释]

**为什么要做？**
[说明目的和价值]

**操作步骤：**

1、[第一步操作]
   > 💡 **说明**：[解释这个操作的作用]

2、[第二步操作]
   > ⚠️ **注意**：[重要提醒]

**验证是否成功**：
```bash
# 验证命令
命令
# 预期输出：xxx
```

**如果失败怎么办**：

**错误1**：`[错误信息]`
**原因**：[原因]
**解决方案**：[方案]
```

### 4.3 添加视觉引导

**必须使用的标记**：
```
💡 提示 - 有用的小技巧
⚠️ 警告 - 重要提醒
❌ 错误做法 - 常见错误示例
✅ 正确做法 - 推荐的方式
📌 信息来源 - 引用来源
🎯 关键点 - 核心要点
```

---

## 阶段5：质量验证

### 5.1 格式检查

```markdown
## 格式检查清单

□ 序号使用1、2、3、格式（不是1. 2. 3.）
□ 没有使用Markdown表格（改用列表）
□ 没有使用---分隔线
□ 代码块标注了语言
□ 所有命令有注释说明
□ 跨平台命令分开写
```

### 5.2 内容检查

```markdown
## 内容检查清单

□ 术语表有10+术语
□ 每个术语有英文全称和生活类比
□ 每个操作解释了"是什么"和"为什么"
□ 每个步骤有💡说明
□ 每个验证方法有预期输出
□ 常见错误都有解决方案
```

### 5.3 信息验证检查

```markdown
## 信息验证检查清单

□ 所有版本号已WebSearch验证
□ 所有URL可正常访问
□ 信息来源已标注
□ 验证日期已标注
```

## 二、快速参考：5分钟检查法

**发布前5分钟快速检查**：

1、✅ 文档顶部有课程信息框？
2、✅ 有术语表？（至少10个术语）
3、✅ 每个安装步骤后有验证命令？
4、✅ 有常见问题章节？
5、✅ 结尾有总结或速查表？
