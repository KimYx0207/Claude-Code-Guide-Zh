# Markdown格式规范

**更新日期**: 2025-12-17
**适用范围**: 直播教程文档写作

---

## 核心要求：飞书兼容性

### 1. 禁止使用`<details>`折叠标签

**原因**：飞书文档无法渲染HTML标签

**错误示例**：
```markdown
<details>
<summary>点击查看答案</summary>
答案是B
</details>
```

**正确做法**：直接显示答案，使用**加粗**和`>`引用块

```markdown
**答案：B**

> **解释**：直接运行`claude`就是启动交互模式。
```

---

### 2. 禁止在代码块第一行使用注释作为说明

**原因**：代码块应该是纯代码，说明应该用Markdown文本

**错误示例**：
````markdown
```bash
# 这是一个脚本说明  ← 不要这样
echo "hello"
```
````

**正确示例**：
````markdown
**脚本说明**：打印hello

```bash
echo "hello"
```
````

## 通用Markdown规范

### 基础格式要求

1. ✅ **使用标准Markdown表格**展示对比信息
2. ✅ **使用 `---` 分隔**不同章节
3. ✅ **代码块必须标注语言**（bash、python、json、powershell等）
4. ✅ **跨平台命令分开写**（Windows/macOS/Linux）
5. ✅ **使用emoji增强可读性**（✅❌💡⚠️🎯📌）
6. ✅ **重要提示用`>`引用块**标注

---

### 代码块格式

**语言标注（小写）**：
- PowerShell命令：`powershell`
- Bash/Shell命令：`bash`
- Python代码：`python`
- JSON配置：`json`
- JavaScript：`javascript`

**示例**：

**Windows系统（PowerShell 7）：**
```powershell
# 查看Node.js版本
node --version
# 预期输出：v22.x.x
```

**macOS/Linux系统：**
```bash
# 查看Node.js版本
node --version
# 预期输出：v22.x.x
```

---

### 视觉引导标记

使用以下标记增强可读性：

```markdown
> 💡 **提示**：有用的小技巧

> ⚠️ **警告**：重要提醒，不注意可能出错

> ❌ **错误做法**：常见错误示例

> ✅ **正确做法**：推荐的方式

> 📌 **信息来源**：[官方文档](URL) | 验证日期：2025-12-XX

> 🎯 **关键点**：核心要点，必须理解
```

---

### 答案展示格式

**知识测试题的答案格式**：

```markdown
**1. 问题描述？**
- A. 选项A
- B. 选项B
- C. 选项C

**答案：B**

> **解释**：详细解释为什么选B。
```

**不要使用**：
- ❌ `<details>`折叠标签（飞书不支持）
- ❌ HTML标签（`<div>`, `<span>`等）
- ❌ 自定义CSS样式
- ❌ 内联HTML（`<br>`, `<hr>`等）

## 表格格式

**对比表格示例**：

```markdown
| 对比项 | 选项A | 选项B | 选项C |
|--------|-------|-------|-------|
| 特点1 | ✅ 支持 | ❌ 不支持 | ⚠️ 部分支持 |
| 特点2 | 快速 | 中等 | 慢 |
| 推荐度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
```

**要求**：
- 表头使用加粗
- 使用emoji增强可读性
- 对齐保持整齐（使用空格）

---

## 步骤格式

**标准步骤格式**：

```markdown
#### 步骤X：[操作名称]

**这一步要做什么**：[一句话说明目的]

**操作方法**：

1. 第一步具体操作
   > 💡 **说明**：解释这个操作的作用

2. 第二步具体操作
   > ⚠️ **注意**：重要提醒

**验证是否成功**：
```bash
# 验证命令
命令
# 预期输出：xxx
```

**如果出错怎么办**：

**错误1**：`错误信息`
**原因**：原因分析
**解决方案**：解决步骤
```

## 跨平台命令格式

**Windows和Unix系统命令分开写**：

```markdown
**Windows系统（PowerShell 7）：**
```powershell
$env:ANTHROPIC_API_KEY
```

**macOS/Linux系统：**
```bash
echo $ANTHROPIC_API_KEY
```
```

**不要混在一起**：
```markdown
❌ 错误做法：
```bash
# Windows
echo %VAR%
# macOS
echo $VAR
```
```

## 信息来源标注

**WebSearch验证后，必须标注来源**：

```markdown
> 📌 **信息来源**：[Node.js官方文档](https://nodejs.org/docs/) | 验证日期：2025-12-17
```

**要求**：
- 包含来源链接
- 标注验证日期
- 使用📌emoji

## 版本号格式

**标准格式**：
```markdown
> - **适用版本**：Node.js v22.x / Claude Code v2.0.71+
```

**说明**：
- 明确主版本号（v22.x表示22系列都可以）
- 如有多个软件，用 `/` 分隔
