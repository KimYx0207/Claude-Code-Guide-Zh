## 第一章：命令系统概述

### 1.1 什么是Slash命令

Slash命令是Claude Code中最核心的交互方式之一。通过在输入框中键入斜杠（/）开头的指令，用户可以快速触发预定义的功能、工作流或自定义脚本。

**命令系统的设计哲学**：

1. **简洁性**：一个斜杠加命令名，即可启动复杂流程
2. **可扩展性**：用户可以创建自定义命令，扩展功能边界
3. **提示词即命令**：每个命令本质上是一个Markdown格式的提示词文件
4. **工具链整合**：命令可以调用MCP工具、Python脚本、Bash命令等

**命令的执行机制**：

当用户输入`/command-name`时，Claude Code会：
1. 在`.claude/commands/`目录中查找对应的`.md`文件
2. 读取文件内容作为系统提示词
3. 将用户的附加参数（如有）作为`$ARGUMENTS`变量传入
4. 根据提示词指引执行相应操作

### 1.2 命令分类体系

Claude Code的命令可以分为两大类：

**内置命令（Built-in Commands）**：
- 由Claude Code官方提供
- 处理会话管理、系统诊断、项目配置等核心功能
- 不可修改，但可以通过自定义命令扩展

**自定义命令（Custom Commands）**：
- 用户在`.claude/commands/`目录创建的命令
- 完全可定制，使用Markdown格式编写
- 可以实现任何复杂的工作流程

### 1.3 命令发现与补全

**Tab补全机制**：
1. 输入`/`后按Tab键，显示所有可用命令
2. 输入部分命令名后按Tab键，自动补全
3. 支持模糊匹配，如`/wr`可以匹配`/write`

**命令搜索**：
- 命令列表按字母顺序排列
- 自定义命令的`name`字段决定显示名称
- `description`字段提供命令说明

## 第二章：内置命令分类详解

Claude Code提供了30+个内置命令，按功能可分为7大类。

### 2.1 会话管理类命令

#### /clear - 清除会话历史

**功能说明**：
清除当前会话的对话历史，重置Claude的上下文记忆。

**使用场景**：
- 切换到完全不同的任务时
- 上下文过长导致响应变慢时
- 需要"干净"的开始时

**语法**：
```
/clear
```

**执行效果**：
- 清除所有对话历史
- 保留CLAUDE.md等项目配置
- 不影响文件系统状态

**最佳实践**：
- 在处理多个独立任务之间使用
- 当Claude开始"忘记"或混淆上下文时使用
- 任务边界明确时主动清理

#### /compact - 压缩会话历史

**功能说明**：
将当前会话历史压缩为更简洁的摘要形式，保留关键信息同时减少token消耗。

**使用场景**：
- 对话过长但不想完全清除
- 需要保留关键决策记录
- 优化token使用效率

**语法**：
```
/compact [可选：保留的关键词或主题]
```

**执行效果**：
- 生成对话摘要
- 删除冗余交互
- 保留核心上下文

**与/clear的区别**：

| 特性 | /clear | /compact |
|------|--------|----------|
| 历史保留 | 完全删除 | 摘要保留 |
| 上下文 | 重置 | 压缩 |
| 适用场景 | 任务切换 | 优化继续 |

#### /resume - 恢复会话

**功能说明**：
从之前保存的会话状态恢复，继续之前的工作。

**使用场景**：
- 重新打开项目时恢复进度
- 意外关闭后恢复上下文
- 跨时间段继续复杂任务

**语法**：
```
/resume [session-id]
```

**执行效果**：
- 加载指定会话的历史
- 恢复工作目录状态
- 重建对话上下文

**注意事项**：
- 需要会话被正确保存
- 可能需要重新验证某些状态
- 长时间未使用的会话可能过期

#### /export - 导出会话

**功能说明**：
将当前会话导出为可分享或存档的格式。

**使用场景**：
- 分享对话内容给团队成员
- 存档重要的技术讨论
- 创建教程或文档素材

**语法**：
```
/export [格式：markdown|json|text]
```

**支持的导出格式**：
- `markdown`：结构化Markdown文件
- `json`：机器可读的JSON格式
- `text`：纯文本格式

#### /sessions - 会话列表

**功能说明**：
查看所有保存的会话列表。

**使用场景**：
- 查找之前的会话
- 管理多个并行项目
- 清理过期会话

**语法**：
```
/sessions [list|delete session-id]
```

**子命令**：
- `list`：列出所有会话（默认）
- `delete`：删除指定会话

### 2.2 内存管理类命令

#### /memory - 记忆管理

**功能说明**：
管理Claude Code的持久化记忆系统，用于存储跨会话的重要信息。

**使用场景**：
- 保存项目偏好设置
- 记录常用配置
- 存储学习到的项目知识

**语法**：
```
/memory [action] [key] [value]
```

**子命令详解**：

**查看所有记忆**：
```
/memory list
```

**添加记忆**：
```
/memory add "key" "value"
```

**删除记忆**：
```
/memory delete "key"
```

**搜索记忆**：
```
/memory search "keyword"
```

**记忆系统架构**：

```
~/.claude/memory/
├── global.json          # 全局记忆
├── projects/
│   ├── project-a.json   # 项目级记忆
│   └── project-b.json
└── sessions/
    └── session-xxx.json  # 会话级记忆
```

**记忆的层级优先级**：
1. 会话级记忆（最高）
2. 项目级记忆
3. 全局记忆（最低）

**最佳实践**：
- 使用有意义的key命名
- 定期清理过期记忆
- 区分临时信息和持久配置

## 第三章：快捷键系统

### 3.1 Escape键 - 快速回退

**功能说明**：
Escape键在Claude Code中具有"时光机"的作用，可以快速回退到之前的状态。

**核心功能**：

**Rewind（回退）**：
- 按Escape键触发
- 回退到上一个交互点
- 撤销Claude的最近响应
- 恢复之前的输入状态

**使用场景**：
1. Claude的回答不满意，想重新提问
2. 意外触发了不想要的操作
3. 需要修改上一个问题的措辞
4. 探索不同的解决方案

**工作原理**：
```
状态1: 用户输入A
  ↓
状态2: Claude响应A'
  ↓
状态3: 用户输入B
  ↓
状态4: Claude响应B'
  ↓
按Escape → 回到状态3
再按Escape → 回到状态1
```

**注意事项**：
- 不能回退已执行的文件操作（使用/undo代替）
- 回退深度有限制
- 某些状态可能无法完全恢复

### 3.2 Tab补全系统

**功能说明**：
Tab键提供智能补全功能，大幅提升输入效率。

**补全类型**：

**命令补全**：
```
/wr[Tab] → /write
/ho[Tab] → /hotspot
/t[Tab]  → 显示所有以t开头的命令
```

**文件路径补全**：
```
Read src/[Tab] → 显示src目录下的文件
Edit articles/[Tab] → 显示articles目录下的文件
```

**Git分支补全**：
```
git checkout fea[Tab] → feature/xxx
```

**补全技巧**：
1. 输入最少字符后按Tab
2. 多次Tab切换候选项
3. Shift+Tab反向切换
4. 直接输入序号选择

**模糊匹配**：
Tab补全支持模糊匹配，例如：
- `wcollect` 可以匹配 `write-collect-data`
- `tgen` 可以匹配 `title-gen`

### 3.3 Ctrl组合键

**Ctrl+C - 中断操作**：
- 中断当前正在执行的操作
- 取消长时间运行的任务
- 停止正在进行的输出

**Ctrl+L - 清屏**：
- 清除终端显示内容
- 不清除会话历史
- 保持当前状态

**Ctrl+D - 退出**：
- 退出Claude Code
- 会提示保存会话
- 优雅关闭连接

**Ctrl+Z - 后台**：
- 将Claude Code放入后台
- 可用`fg`命令恢复
- Unix/Linux系统有效

**Ctrl+R - 搜索历史**：
- 搜索命令历史
- 支持正则匹配
- 快速重复命令

**Ctrl+A/E - 行首/行尾**：
- Ctrl+A：光标移到行首
- Ctrl+E：光标移到行尾
- 标准终端快捷键

**Ctrl+W - 删除单词**：
- 删除光标前的一个单词
- 快速清理输入
- 标准终端行为

**Ctrl+U - 删除到行首**：
- 删除从光标到行首的所有内容
- 快速清空输入
- 保留行尾内容

## 第四章：命令作用域

### 4.1 作用域概念

Claude Code的命令系统采用层级作用域设计，不同级别的命令有不同的生效范围和优先级。

**作用域层级**：

```
全局作用域 (Global)
    ↓
用户作用域 (User)
    ↓
项目作用域 (Project)
    ↓
会话作用域 (Session)
```

### 4.2 Project级命令

**定义位置**：
```
<project-root>/.claude/commands/
```

**特点**：
- 仅在当前项目中可用
- 可以提交到版本控制
- 团队成员共享
- 与项目配置绑定

**典型用例**：
- 项目特定的构建命令
- 团队约定的工作流
- 项目相关的自动化脚本

**示例结构**：
```
my-project/
└── .claude/
    └── commands/
        ├── build.md         # 项目构建
        ├── deploy.md        # 项目部署
        ├── test.md          # 项目测试
        └── lint.md          # 代码检查
```

### 4.3 User级命令

**定义位置**：
```
~/.claude/commands/
```

**特点**：
- 在所有项目中可用
- 个人偏好设置
- 不随项目传播
- 跨项目复用

**典型用例**：
- 个人效率工具
- 通用工作流模板
- 个人习惯的快捷方式

**示例结构**：
```
~/.claude/
└── commands/
    ├── my-template.md      # 个人模板
    ├── quick-search.md     # 快速搜索
    └── daily-routine.md    # 日常流程
```

### 4.4 优先级机制

当同名命令存在于多个作用域时，按以下优先级解析：

**优先级顺序（从高到低）**：
1. **Project级**：`.claude/commands/`
2. **User级**：`~/.claude/commands/`
3. **内置命令**：Claude Code自带

**优先级示例**：

假设存在以下命令定义：
```
内置: /review
~/.claude/commands/review.md
.claude/commands/review.md
```

执行`/review`时：
- 使用 `.claude/commands/review.md`（项目级优先）

**强制指定作用域**：
```
/project:review    # 强制使用项目级
/user:review       # 强制使用用户级
/builtin:review    # 强制使用内置
```

### 4.5 作用域最佳实践

**Project级适用场景**：
- 团队协作规范
- 项目特定流程
- CI/CD集成命令
- 项目文档生成

**User级适用场景**：
- 个人效率工具
- 跨项目通用模板
- 个人偏好快捷方式
- 实验性命令

**命名规范建议**：
```
# Project级：项目相关前缀
/myapp-build
/myapp-deploy

# User级：个人标识前缀
/my-template
/quick-xxx
```

**版本控制策略**：
```
# .gitignore 建议配置
.claude/commands/local-*.md    # 本地命令不提交
.claude/settings.local.json    # 本地配置不提交
```

## 第六章：命令最佳实践

### 6.1 命令使用原则

**1. 最小惊讶原则**：
- 命令行为应符合直觉
- 避免副作用不明确的操作
- 破坏性操作应有确认

**2. 渐进式深入**：
- 从简单命令开始
- 逐步学习高级用法
- 按需探索专业功能

**3. 组合优于复杂**：
- 小命令组合完成复杂任务
- 避免创建过于复杂的单一命令
- 利用管道和链式调用

### 6.2 常见问题解决

**Q1: 命令不存在**
```
检查步骤：
1. 确认命令名称拼写
2. 检查命令文件是否存在
3. 验证作用域配置
4. 重启Claude Code
```

**Q2: 命令执行失败**
```
检查步骤：
1. 查看错误信息
2. 验证依赖工具可用
3. 检查权限配置
4. 查看日志文件
```

**Q3: 命令效果不符预期**
```
检查步骤：
1. 确认参数传递正确
2. 检查$ARGUMENTS处理
3. 验证前置条件
4. 查看命令源文件
```

### 6.3 效率提升技巧

**1. 建立命令别名**：
为常用长命令创建短名称版本

**2. 使用命令历史**：
Ctrl+R搜索历史命令

**3. 批量操作**：
将重复任务封装为命令

**4. 模板化工作流**：
创建可参数化的命令模板

**5. 定期清理**：
移除不再使用的自定义命令
